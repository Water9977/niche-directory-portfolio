# Workspace Rules — Niche Directory Portfolio

## Planning-First Protocol
- **NO CODE** is to be written until the user explicitly says to start building.
- All design decisions, schemas, and feature specs must be documented in `architecture.md` FIRST.
- `architecture.md` is the single source of truth for the entire project.

## Architecture-First Updates
- Before modifying any feature or adding any file, update `architecture.md` to reflect the change.
- After implementation, update `architecture.md` to mark the feature as complete.

## Commit Discipline
- Every meaningful change gets its own commit with a descriptive message.
- Use conventional commit prefixes: `docs:`, `feat:`, `fix:`, `chore:`, `refactor:`.
